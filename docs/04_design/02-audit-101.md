# Audit 101 — AI Provider & Agents SDK Inconsistencies

**Date:** 2026-06-29
**Scope:** AI model provider routing, Agents SDK usage, LLM calling patterns
**Target State:** OpenRouter (auto-free models) as primary → Cloudflare Workers AI as fallback. No Groq.

## 1. Provider Routing — No Multi-Provider Abstraction

Every LLM call hardcodes a single provider. No routing layer selects a provider based on availability, cost, or configuration.

| File | Hardcoded Provider | Lines |
|------|-------------------|-------|
| `apps/api/agent/src/agents/TajiAgent.ts` | Groq (primary), Workers AI (fallback) | 293–303, 305–313 |
| `apps/api/agent/src/agents/ElimAgent.ts` | Groq (primary), Workers AI (fallback) | 39–45 |
| `apps/api/agent/src/lib/groq.ts` | Groq only | 16–48 |
| `apps/api/docgen/src/pipeline/extractor.ts` | Groq only | 28–179 |
| `apps/api/docgen/src/pipeline/converters/docx-to-markdown.ts` | Groq only | 12–44 |
| `apps/api/docgen/src/pipeline/converters/docx-to-schema.ts` | Groq only | 17–38 |
| `apps/api/docgen/src/pipeline/converters/pdf-to-schema.ts` | Groq only | 20–38 |
| `apps/api/docgen/src/pipeline/converters/vision-to-schema.ts` | Workers AI (vision) + Groq (schema) | 23, 41 |

**OpenRouter has zero references in the entire codebase.**

**Solution:** Create a unified `LLMService` that accepts a provider name, resolves the endpoint and API key, falls through on failure: OpenRouter → Workers AI.

## 2. Provider Type Definitions Mismatch

Types and docs list `'groq' | 'cloudflare-ai' | 'openai'` but OpenRouter is absent and OpenAI is unimplemented.

| Location | Defined Providers | Status |
|----------|-----------------|--------|
| `packages/types/src/index.ts:75` | `'groq' \| 'cloudflare-ai' \| 'openai'` | Missing OpenRouter |
| `apps/api/agent/src/db/schema.ts:10` | Default `'groq'` | Same missing types |
| `docs/agents/agent-model.md:17` | `'groq' \| 'cloudflare_ai' \| 'openai'` | Uses underscore variant |
| `docs/architecture/platform.md:29` | `"groq" \| "cloudflare_ai" \| "openai"` | Same underscore variant |

**Solution:** Replace with `'openrouter' | 'workers-ai'` in types, drop Groq and OpenAI, fix hyphen/underscore inconsistency.

## 3. Inconsistent LLM Calling Patterns Between Agents

| Aspect | TajiAgent | ElimAgent |
|--------|-----------|-----------|
| API Key source | `agentConfig.groqApiKey ?? env.GROQ_API_KEY` | `env.GROQ_API_KEY` only |
| Model selection | `agentConfig.modelId ?? 'llama-3.3-70b-versatile'` | Hardcoded `'llama-3.3-70b-versatile'` |
| System prompt | `agentConfig.systemPrompt ?? TAJI_SYSTEM_PROMPT` | Hardcoded `ELIM_PROMPT` constant |
| Max tokens | 512 | 768 |
| Temperature | 0.7 | 0.5 |
| Fallback | `cfAI()` method (Workers AI) | Inline Workers AI call |

**Solution:** Both agents use a shared `LLMService` class backed by OpenRouter API. Agent config becomes provider-agnostic.

## 4. Workers AI Binding Type Safety

All Workers AI calls bypass TypeScript with `as any` casts across 3 files.

**Solution:** Create a typed Workers AI client wrapper with a model registry.

## 5. Agent SDK Version & Configuration Drift

`agents` SDK v0.0.97 installed but `tsconfig.json` previously extended non-existent `agents/tsconfig`, `hono-agents` v0.0.20 uses `agentsMiddleware` but code was written for `honoAgents` API.

**Solution:** Pin SDK versions, document which API pattern is used.

## 6. No Provider Failover or Circuit-Breaker

Current fallback is a single try/catch. No retry with backoff, no circuit-breaker, no rate-limit detection.

**Solution:** Two-tier provider chain: OpenRouter first (configurable timeout, retry), Workers AI fallback.

## 7. ConversationMachine Has No LLM Access

The machine is a pure state machine that must delegate all LLM calls to Agent DOs via HTTP fetch.

**Solution:** Give the machine access to `LLMService` for lightweight AI decisions (intent classification, entity extraction).

## 8. Environment Variables & Configuration Gaps

Only `GROQ_API_KEY` exists. Missing: `OPENROUTER_API_KEY`, `OPENROUTER_BASE_URL`, `AI_PROVIDER`, `AI_MODEL`.

**Solution:** Replace `GROQ_API_KEY` with `OPENROUTER_API_KEY` across all config files, env types, and wrangler.toml.

## 9. No Provider URL Abstraction — Groq URLs Leaked Everywhere

Groq API endpoints are hardcoded in **5 separate files**.

**Solution:** Delete the Groq-specific code path entirely. `LLMService` owns the single OpenRouter endpoint.
